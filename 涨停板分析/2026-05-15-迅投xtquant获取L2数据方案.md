# 迅投 xtquant 获取L2行情数据方案

**来源**: QQ | **日期**: 2026-05-15

## 文档背景

在研究如何获取A股Level2（逐笔成交、十档盘口）数据时，发现迅投（ThinkTrader）的 xtquant 库提供了完整的L2数据接口。本文档记录该方案的详细分析。

## 数据能力一览

| 接口名 | 数据内容 | L2等级 |
|:---|:---|---:|
| `l2transaction` | **逐笔成交** — 每一笔买卖的时间、价格、量、方向 | ✅ 真L2 |
| `l2order` | **逐笔委托** — 每一笔挂单的详情（价格、量、时间） | ✅ 真L2 |
| `l2quote` | **十档行情快照** — 买一~买十 / 卖一~卖十的挂单详情 | ✅ 真L2 |
| `l2orderqueue` | **一档委托队列** — 买一/卖一的前50笔挂单明细 | ✅ 真L2 |
| `l2quoteaux` | 总买总卖量（全市场委托汇总） | ✅ 真L2 |
| tick | 分笔成交数据（比K线更细但不如L2逐笔） | ✅ 准L2 |
| 1m / 5m / 1d | 标准K线数据 | ✅ 基础 |
| 财务数据 | 利润表、资产负债表、现金流量表、主要指标、股本表 | ✅ |
| 板块/行业 | 获取板块列表、成分股 | ✅ |

## 架构原理

```
你的Python策略脚本
    ↕ (Python调用)
xtquant Python库 (pip install xtquant)
    ↕ (本地IPC通信)
MiniQMT 客户端 ← Windows桌面软件，必须保持运行
    ↕ (连接券商行情服务器)
交易所行情源
```

核心逻辑：xtquant 不做独立的数据存储，它通过本地 IPC 连接到 MiniQMT 客户端，由 MiniQMT 从券商服务器拉取数据后转发给 Python 脚本。

## 使用条件

### 必要条件

1. **Windows 操作系统** — MiniQMT 只有 Windows 版本，不支持 Linux/macOS
2. **QMT 券商账户** — 需要找券商开通 QMT 量化交易权限（如银河证券、国金证券、华泰证券、中信证券等）
3. **安装 MiniQMT 客户端** — 从券商处获取并登录
4. **安装 xtquant Python 库** — `pip install xtquant`
5. **Python 3.6 ~ 3.12（64位）** — 均可支持

### 使用步骤

```python
# 1. 先启动 MiniQMT 客户端并保持运行
# 2. 安装 xtquant
# pip install xtquant

from xtquant import xtdata

# 连接 MiniQMT（需先启动客户端）
# xtdata 会自动发现本机的 MiniQMT 进程

# 获取历史行情（本地行情，无需实时订阅）
# 例如获取日线数据
data = xtdata.get_market_data_ex(
    field_list=['time', 'open', 'high', 'low', 'close', 'volume', 'amount'],
    stock_list=['600276.SH'],
    period='1d',
    start_time='20250501',
    end_time='20260515',
    count=5
)

# 获取K线数据
data = xtdata.get_local_data(
    field_list=[],
    stock_list=['600276.SH'],
    period='1m',
    start_time='20260514',
    end_time='20260514'
)

# 下载历史行情数据（先下载到本地）
xtdata.download_history_data(
    stock_code='600276.SH',
    period='1d',
    start_time='20250101',
    end_time='20260515'
)

# 下载L2逐笔成交数据
xtdata.download_history_data(
    stock_code='600276.SH',
    period='l2transaction',
    start_time='20260514',
    end_time='20260514'
)

# 下载L2逐笔委托数据
xtdata.download_history_data(
    stock_code='600276.SH',
    period='l2order',
    start_time='20260514',
    end_time='20260514'
)

# 获取已下载的L2逐笔成交数据
transactions = xtdata.get_local_data(
    field_list=[],
    stock_list=['600276.SH'],
    period='l2transaction',
    start_time='20260514',
    end_time='20260514'
)
```

### 实时订阅（盘中）

```python
from xtquant import xtdata
import time

# 定义回调函数
def my_callback(data):
    print(f"收到行情更新: {data}")

# 订阅单股行情
xtdata.subscribe_quote(
    stock_code='600276.SH',
    period='tick',  # 实时分笔
    callback=my_callback
)

# 订阅L2实时行情快照
xtdata.subscribe_quote(
    stock_code='600276.SH',
    period='l2quote',
    callback=my_callback
)

# 保持线程接收
xtdata.run()
```

## L2逐笔成交字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `time` | int | 成交时间（毫秒级时间戳） |
| `price` | float | 成交价 |
| `volume` | int | 成交量（股） |
| `turnover` | float | 成交额（元） |
| `buy_orsell` | int | 成交方向（1=买, 2=卖） |
| `function_code` | int | 成交标志 |

## L2逐笔委托字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `time` | int | 委托时间 |
| `price` | float | 委托价格 |
| `volume` | int | 委托量（股） |
| `side` | int | 委托方向（1=买, 2=卖） |
| `order_kind` | int | 委托类型 |

## L2十档行情字段说明

| 字段 | 说明 |
|:---|:---|
| `bid_price` | 买一到买十价格数组 |
| `bid_volume` | 买一到买十数量数组 |
| `ask_price` | 卖一到卖十价格数组 |
| `ask_volume` | 卖一到卖十数量数组 |
| `last_price` | 最新成交价 |
| `open` / `high` / `low` / `close` | 开盘/最高/最低/收盘 |
| `volume` | 成交量 |
| `amount` | 成交额 |

## 与现有数据源的对比

| 对比维度 | 现有方案(Tushare + 东财) | xtquant + MiniQMT |
|:---|:---|:---|
| **逐笔成交** | ❌ 不可用 | ✅ ❤️ 完整逐笔 |
| **十档盘口** | ❌ 不可用 | ✅ 买一~买十 |
| **逐笔委托** | ❌ 不可用 | ✅ 完整逐笔 |
| **历史数据** | ✅ 丰富 | ✅ 完整 |
| **盘中实时** | ✅ 分钟K线级别 | ✅ 毫秒级tick |
| **运行平台** | ✅ Linux服务器 | ❌ 需要Windows |
| **费用** | Tushare积分 | 需要QMT账户(券商入金开通) |
| **安装复杂度** | ✅ 简单 | ⚠️ 需要安装客户端 |
| **数据覆盖面** | 全A股+宏观+板块+资金流 | 全A股+港股+期货 |

## 关键结论

**xtquant 是目前能接触到的最完整的A股L2数据方案**，包含了逐笔成交、逐笔委托、十档行情等真正的L2数据。

**但局限性也很明显：**
1. 需要**Windows系统**运行MiniQMT客户端 — Linux服务器用不了
2. 需要**开通券商QMT权限** — 需要有一定资金门槛
3. MiniQMT客户端必须**保持运行**才能获取数据

**最佳使用场景：** 在自己Windows电脑上搭建本地量化环境，策略跑在本地，xtquant提供实时L2行情数据。
